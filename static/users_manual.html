üìö Getting Started

Welcome to CalypsoPy+

CalypsoPy+ is a professional PCIe hardware development interface designed for interfacing with Serial Cables Gen6 PCIe Atlas 3 Host Cards. This comprehensive web-based dashboard provides real-time hardware monitoring, firmware update capabilities, and extensive testing functionality for PCIe devices.

Professional Tool: CalypsoPy+ is designed for hardware engineers, validation teams, and PCIe development professionals working with Atlas 3 hardware.
üì∏
Screenshot: Main dashboard overview showing all available modules

Quick Start Checklist

  1. Verify System Requirements: Ensure your system meets the minimum requirements
  2. Install Dependencies: Run python start.py to automatically check and install required packages
  3. Connect Hardware: Connect your Atlas 3 device via serial cable
  4. Launch Application: Start CalypsoPy+ and navigate to http://localhost:5000
  5. Establish Connection: Configure serial connection parameters and connect to device
  6. Verify Connection: Check device information dashboard for successful communication
Quick Tip: Use Ctrl+F to search this manual for specific topics or commands.

Device Connection

CalypsoPy+ communicates with Atlas 3 hardware through serial communication. Proper connection setup is essential for reliable operation.

Connection Process

  1. Physical Connection: Connect the serial cable between your computer and the Atlas 3 device
  2. Port Detection: The application will automatically scan for available serial ports
  3. Parameter Configuration: Set connection parameters (typically auto-detected)
  4. Connection Establishment: Click "Connect" to establish communication
  5. Verification: Confirm successful connection through status indicators

üîÄ Bifurcation

Port Configuration

The Bifurcation dashboard allows configuration of PCIe port modes and lane distribution across the Atlas 3 device. This feature enables flexible adaptation to different hardware configurations and use cases.

Understanding Bifurcation

PCIe bifurcation allows a single physical PCIe slot to be divided into multiple logical slots with fewer lanes each. For example, a x16 slot can be bifurcated into:

  • x16: Single port with all 16 lanes
  • x8+x8: Two ports with 8 lanes each
  • x8+x4+x4: One 8-lane and two 4-lane ports
  • x4+x4+x4+x4: Four 4-lane ports
üì∏
Screenshot: Bifurcation configuration interface showing port layout and mode selection

Mode Explanations

Atlas 3 supports multiple bifurcation modes optimized for different scenarios:

Standard Modes

Mode Configuration Use Case Performance
Mode 0 x16 Single high-speed device Maximum bandwidth
Mode 1 x8+x8 Two high-speed devices Balanced performance
Mode 2 x8+x4+x4 Mixed device speeds Optimized allocation
Mode 3 x4+x4+x4+x4 Multiple standard devices Maximum device count

Advanced Modes

Additional modes are available for specialized applications:

  • Custom Mode: User-defined lane allocation
  • Test Mode: Optimized for validation and testing
  • Debug Mode: Enhanced monitoring and logging
  • Failsafe Mode: Conservative settings for stability
Mode Changes: Changing bifurcation mode requires a device reset and may affect connected devices. Plan changes during maintenance windows.

üîÑ Firmware Updates

XMODEM-Based Update Process

CalypsoPy+ uses the robust XMODEM protocol for firmware updates, ensuring reliable file transfer even over noisy serial connections. The update process is designed for safety and includes comprehensive error checking and recovery mechanisms.

Update Process Overview

  1. Pre-Update Validation: Verify firmware file integrity and compatibility
  2. Device Preparation: Put target device into update mode
  3. XMODEM Transfer: Reliable file transfer with error correction
  4. Verification: Confirm successful update and integrity
  5. Reset and Validation: Restart device and verify operation
CRITICAL: Never interrupt firmware updates or power cycle the device during update. Interruption may render the device inoperable and require factory recovery.
üì∏
Screenshot: Firmware update interface showing target selection, file selection, and progress indicators

Update Targets

Atlas 3 devices contain multiple programmable components, each with its own firmware update process:

MCU Firmware

  • Function: Main control unit managing device operation
  • Update Method: XMODEM over primary serial connection
  • File Format: Intel HEX (.hex) or binary (.bin)
  • Update Time: Typically 2-5 minutes
  • Recovery: Hardware-based recovery if update fails

SBR0 Firmware (Side-band Controller 0)

  • Function: First side-band management controller
  • Update Method: XMODEM via MCU relay
  • File Format: Binary (.bin) files only
  • Update Time: Typically 1-3 minutes
  • Dependencies: Requires compatible MCU firmware

SBR1 Firmware (Side-band Controller 1)

  • Function: Second side-band management controller
  • Update Method: XMODEM via MCU relay
  • File Format: Binary (.bin) files only
  • Update Time: Typically 1-3 minutes
  • Dependencies: Requires compatible MCU firmware
Target File Types Max Size Update Command
MCU .hex, .bin 2 MB update mcu
SBR0 .bin 1 MB update sbr0
SBR1 .bin 1 MB update sbr1

Screenshot: Firmware update progress dialog showing progress bars, statistics, and status messages

Update Phases

The firmware update process consists of several distinct phases:

  1. Initialization: Prepare device and validate firmware file
  2. Handshake: Establish XMODEM communication with target
  3. Transfer: Send firmware data packets with error checking
  4. Verification: Verify transferred data integrity
  5. Finalization: Complete update and restart target
  6. Validation: Confirm successful operation of new firmware

Error Handling

Robust error handling ensures reliable updates:

  • Automatic Retry: Failed packets automatically retransmitted
  • Checksum Validation: Every packet verified for integrity
  • Timeout Management: Appropriate timeouts for each phase
  • Graceful Failure: Safe failure modes that preserve device

Post-Update Verification

After successful update, the system performs comprehensive verification:

  • Version Verification: Confirm new firmware version
  • Functionality Test: Basic operational tests
  • Communication Test: Verify device communication
  • Configuration Preservation: Ensure settings retained
Update Success: Successful updates are automatically logged with before/after version information and validation results.
# Example: Firmware update command sequence update mcu firmware_v2.1.3.hex # Start MCU update # Progress monitoring automatically displayed # Verification performed upon completion getversion # Verify new version # MCU: v2.1.3, SBR0: v1.5.2, SBR1: v1.5.2

Recovery Procedures

In case of update failures, several recovery options are available:

  • Retry Update: Attempt update again with same firmware
  • Alternative Firmware: Try older or different firmware version
  • Hardware Recovery: Use dedicated recovery pins/jumpers
  • Factory Recovery: Return device to manufacturer for recovery
Recovery Mode: If device becomes unresponsive after failed update, refer to hardware recovery procedures in the troubleshooting section.

Common Issues and Solutions

This section covers the most frequently encountered issues and their solutions when using CalypsoPy+ with Atlas 3 hardware.

Connection Issues

Device Not Detected

Symptoms: No serial ports detected or device not responding

Solutions:

  1. Verify physical serial cable connection
  2. Check device power status and indicators
  3. Restart CalypsoPy+ application
  4. Try different USB/serial port
  5. Verify driver installation for USB-to-serial adapters
Communication Timeouts

Symptoms: Commands time out or return "No response" errors

Solutions:

  1. Check serial connection parameters (115200-8-N-1)
  2. Verify device is not in firmware update mode
  3. Increase timeout values in connection settings
  4. Reset device using hardware reset button
  5. Check for electromagnetic interference
Intermittent Connection Drops

Symptoms: Connection established but drops randomly

Solutions:

  1. Check cable quality and connections
  2. Verify stable power supply to device
  3. Disable aggressive power management on host system
  4. Update USB/serial drivers
  5. Try different cable or serial adapter
üì∏
Screenshot: Connection troubleshooting interface showing diagnostic tools and status

Firmware Update Issues

Update Fails to Start

Symptoms: Firmware update process won't initiate

Solutions:

  1. Verify firmware file format and integrity
  2. Check device is in normal operation mode
  3. Ensure no other applications are using serial port
  4. Restart device and retry update
  5. Verify sufficient power supply to device
Update Interrupted

Symptoms: Update process stops mid-transfer

Solutions:

  1. DO NOT POWER CYCLE DEVICE
  2. Retry update immediately
  3. If retry fails, attempt recovery mode
  4. Check hardware recovery procedures
  5. Contact support if device becomes unresponsive
CRITICAL: Never power cycle device during firmware update. This may render the device inoperable.

Performance Test Issues

Tests Fail to Run

Symptoms: Performance tests won't execute or fail immediately

Solutions:

  1. Verify administrator/root privileges
  2. Check FIO installation and accessibility
  3. Ensure target devices are accessible
  4. Verify no other applications are using test devices
  5. Check available disk space for test files
Inconsistent Results

Symptoms: Performance results vary significantly between runs

Solutions:

  1. Ensure system is idle during testing
  2. Disable background applications and services
  3. Check thermal throttling on devices
  4. Verify consistent power delivery
  5. Allow device to cool between test runs

Emergency Recovery Procedures

Procedures for recovering from critical failures or device lockups:

Software Recovery

Application Crashes
  1. Check log files in logs/calypso_py.log
  2. Restart CalypsoPy+ application
  3. If persistent, delete cache files and restart
  4. Reinstall application if crashes continue
Device Communication Lost
  1. Attempt to reconnect through application
  2. Close application and restart
  3. Power cycle Atlas 3 device
  4. If unsuccessful, proceed to hardware recovery

Hardware Recovery

Device Unresponsive

WARNING: Use only as last resort when device does not respond to software commands

  1. Locate hardware reset button on Atlas 3 device
  2. Hold reset button for 10 seconds while powered
  3. Release reset button and wait 30 seconds
  4. Attempt to reconnect through CalypsoPy+
  5. If still unresponsive, contact technical support
Firmware Recovery Mode

For devices with corrupted firmware:

  1. Power off device completely
  2. Locate recovery jumper on device (refer to hardware manual)
  3. Install recovery jumper
  4. Power on device (should enter recovery mode)
  5. Attempt firmware update with recovery firmware
  6. Remove recovery jumper after successful update
Recovery Mode: Recovery procedures may void warranty. Contact support before attempting hardware recovery.
